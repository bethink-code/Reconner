# Pieter's Pomp Stasie Reconner
## Sequential File Processing UX Specification

---

## 1. Executive Summary

### The Problem
The current UX presents users with three file uploads simultaneously, creating cognitive overload. Users don't know:
- Which file to start with
- What each file is for
- Whether they're making progress
- What happens if one file has issues

### The Solution
A **sequential wizard** that guides users through one file at a time, with clear context about why each file matters and what to do at each step.

### Key Principle
> "One file. One focus. Complete it. Move on."

---

## 2. Overview of the Flow

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│   ┌─────────┐      ┌─────────┐      ┌─────────┐      ┌─────────┐       │
│   │ STEP 1  │ ──▶  │ STEP 2  │ ──▶  │ STEP 3  │ ──▶  │  DONE   │       │
│   │  Fuel   │      │ Bank #1 │      │ Bank #2 │      │ Review  │       │
│   │  Data   │      │  (FNB)  │      │ (ABSA)  │      │ & Match │       │
│   └─────────┘      └─────────┘      └─────────┘      └─────────┘       │
│                                                                         │
│   Source of        Verification     Additional        Run               │
│   Truth            Data             Verification      Reconciliation    │
│                                     (Optional)                          │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### Each Step Contains Sub-Steps

```
For each file:
   1. Upload        → Drag/drop or browse for file
   2. Quality Check → Auto-detect and fix issues (page breaks, headers, etc.)
   3. Column Map    → Match columns to standard fields
   4. Preview       → Confirm data looks correct
   5. Complete      → Move to next step
```

## 5. Key UX Principles

### 5.1 One Thing at a Time
- Never show multiple file uploads simultaneously
- Complete one file fully before moving to the next
- Clear visual progress indicator

### 5.2 Context at Every Step
- Explain WHY this file matters
- Show how it connects to previous steps
- Indicate what comes next

### 5.3 Reassuring Language
- "Source of truth" not "primary file"
- "Verification data" not "secondary file"  
- "We'll fix this automatically" not "Error detected"

### 5.4 Smart Defaults
- Pre-select column mappings based on analysis
- Remember bank selection for appropriate defaults
- Show sample values to validate mappings

### 5.5 Progressive Disclosure
- Show summary first, details on expand
- Don't overwhelm with technical information
- Explain issues in plain language

### 5.6 Clear Escape Hatches
- "Choose different file" always available
- "Skip" option for optional steps
- "Back" to previous step/sub-step

---

## 6. Edge Cases

### 6.1 User Wants to Change Step 1 After Completing Step 2
- Allow going back, but warn: "Changing fuel data will reset your bank file mappings"
- Offer to keep bank files uploaded but re-run coverage analysis

### 6.2 Bank File Has No Overlapping Dates
- Show clear warning with specific dates
- Still allow proceeding (maybe they'll upload more data later)
- Don't fail silently — make the coverage gap obvious

### 6.3 User Only Has One Bank Account
- "Skip" on Step 3 goes directly to completion
- Completion screen adjusts messaging accordingly

### 6.4 User Uploads Same File Twice
- Detect duplicate files
- "This looks like the same file as Bank #1. Upload a different file or skip this step."

### 6.5 Very Large Files
- Show upload progress
- Process in background with progress indicator
- Don't block UI during processing
