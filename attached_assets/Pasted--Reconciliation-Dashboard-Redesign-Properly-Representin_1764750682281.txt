# Reconciliation Dashboard Redesign
## Properly Representing Verifiable vs Unverifiable Data

**Core Principle:** The fuel system is the SOURCE OF TRUTH. Bank data lets us VERIFY a portion of it.

---

## 🎯 Current UI Problems

### Problem 1: Misleading Metrics
Current UI shows "3.81% matched" which makes users think the system is broken.

**Reality:** 79.4% of verifiable transactions matched. System works great!

### Problem 2: Wrong Mental Model
```
Current UI implies: "Match all fuel to bank"
Correct model: "Verify what we can, flag what we can't"
```

### Problem 3: No Visibility into Coverage
Users can't see:
- How much CAN be verified?
- How much is unverifiable (missing bank data)?
- Where are the gaps?

---

## 📊 Key Improvements

### 1. Correct Mental Model
```
OLD: "Match everything" (impossible with 5% coverage)
NEW: "Verify what we can, flag what we can't" (realistic)
```

### 2. Proper Metrics
```
OLD: "3.81% match rate" (misleading - includes unverifiable)
NEW: "79.4% of verifiable transactions matched" (accurate)
     "94.9% awaiting bank data" (honest about coverage)
```

### 3. Actionable Insights
```
OLD: Shows numbers without context
NEW: 
- "Upload missing bank statements" (critical)
- "Review 119 unmatched" (important)
- "Your performance is excellent" (reassurance)
```

### 4. Transparency About Coverage
```
Users clearly see:
- What CAN be verified: 579 bank transactions
- What CANNOT be verified: 11,678 (no bank data yet)
- Why gaps exist: missing accounts, incomplete dates
- What to do: upload additional statements
```
