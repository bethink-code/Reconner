Completion State Logic
Developer Specification
Fix: Don't show "All Done!" when items are flagged for review
The Problem
CURRENT BUG
"All Done!" displays regardless of how transactions were resolved.
If user flagged items for review, they're NOT done — work is pending.
This misleads users into thinking reconciliation is complete when it isn't.
Resolution Types & Their Status
Resolution Action	Meaning	Status
Linked	Matched to fuel record	✓ CLOSED
Marked as Reviewed	Investigated, no issue found	✓ CLOSED
Dismissed	Low value / test transaction	✓ CLOSED
Flagged for Review	Handed off for investigation	⚠️ PENDING
Key Insight: "Flagged for Review" is NOT closed. It's deferred work.
 
Logic
Data Structure
interface ResolutionSummary {
  linked: number;      // Matched to fuel record
  reviewed: number;    // Marked as reviewed
  dismissed: number;   // Low value dismissed
  flagged: number;     // Flagged for review ← KEY FIELD
}
Decision Logic
// Calculate totals
const closedCount = linked + reviewed + dismissed;
const pendingCount = flagged;

// Decide which UI to show
if (pendingCount === 0) {
  // All truly done - show success state
  showAllDoneState();
} else {
  // Work still pending - show summary state
  showReviewCompleteState(summary);
}
 
UI States

Show this ONLY when all transactions are CLOSED (no flagged items):

Show this when ANY transactions are FLAGGED:

Acceptance Criteria
1.	"All Done!" ONLY shows when flagged === 0
2.	"Review Complete" shows when flagged > 0
3.	Summary breakdown shows count per resolution type
4.	Flagged count highlighted with warning styling (orange)
5.	[View Flagged Items] button navigates to filtered Investigation View
6.	[Export for Review] generates Excel with flagged items only
7.	Resolution counts are accurate and match database totals

TL;DR
if (flagged === 0) → "All Done!"
if (flagged > 0)  → "Review Complete" + summary + actions
